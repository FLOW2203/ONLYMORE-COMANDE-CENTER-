## Convention

- **Nom de fichier** : kebab-case, sans préfixe date (le runbook est vivant, daté en frontmatter)
- **Statut** : `draft` → `v1.0` → versions ultérieures (`v1.1`, `v2.0`)
- **Validation** : un runbook passe en `v1.0` après première exécution réussie documentée
- **Référence croisée** : chaque runbook cite la ou les PR / issues / docs source en tête de fichier
